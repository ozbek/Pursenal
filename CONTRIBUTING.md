## Getting Started

1. **Fork the Repository:**  
   Create a personal fork of the Pursenal repository.

2. **Clone Your Fork:**  

   ```bash
   git clone https://github.com/Kaashier-Dev/Pursenal.git
   cd Pursenal
   ```

3. **Create a Branch:**  
   Create a new branch for your feature or bug fix:

   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Install Dependencies:**  
   Make sure you have the latest dependencies by running:

   ```bash
   flutter pub get
   ```

## How to Contribute

- **Small Changes:**  
  For minor improvements or bug fixes, feel free to make a commit on your branch.

- **Major Features:**  
  If you're planning on significant changes, open an issue or start a discussion first to gather feedback. This helps coordinate efforts and ensures your work aligns with the project’s vision.

- **Documentation:**  
  If you improve documentation, make sure the updates are clear and consistent with the rest of the project.

## Reporting Bugs and Feature Requests

Before reporting a bug or suggesting a feature, please:

- Search existing [issues](https://github.com/Kaashier-Dev/Pursenal/issues) to see if it’s already been reported.
- Provide clear, detailed information including steps to reproduce a bug, screenshots, and any relevant logs.
- For feature requests, explain the problem you’re solving and why it would benefit the community.

## Pull Request Guidelines

When you’re ready to submit your changes:

- **Describe Your Changes:**  
  Provide a clear summary of the changes in your PR description, and reference any related issues (e.g., “Fixes #123”).

- **Follow the Code Style:**  
  Make sure your code is clean, well-commented, and follows the existing style. Consistency is key!

- **Include Tests:**  
  Whenever possible, add tests to verify your changes. This helps maintain the integrity of the codebase.

- **One Feature per PR:**  
  Keep your pull requests focused on a single issue or feature to simplify the review process.

## Coding Guidelines

- **Readability:**  
  Write clear and concise code with proper comments where needed.
  
- **Modularity:**  
  Ensure that your changes are modular and respect the separation of concerns (e.g., UI, business logic, and database interactions are kept separate).

- **Documentation:**  
  Update relevant documentation whenever you add or change functionality.
